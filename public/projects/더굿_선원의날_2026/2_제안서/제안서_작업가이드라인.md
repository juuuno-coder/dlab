## 4. 작업 방법론 (재사용 가능한 지침)

### A. PPTX 분석 프로세스

1. **텍스트 추출**: `python -m markitdown 파일.pptx` → 전체 슬라이드 텍스트 확인
2. **시각적 확인**: `soffice --headless --convert-to pdf` → `pdftoppm` → 개별 슬라이드 이미지 생성
3. **슬라이드 맵 작성**: 각 슬라이드의 섹션/제목/상태를 표로 정리
4. **XML 분석** (필요시): `python scripts/office/unpack.py` 로 언팩 후 개별 slide XML 확인

### B. 슬라이드 콘텐츠 편집 (텍스트 수정)

1. **언팩**: `python scripts/office/unpack.py input.pptx unpacked/`
2. **XML 직접 편집**: Edit 도구로 `unpacked/ppt/slides/slideN.xml` 수정
3. **규칙**:
   - 한글 텍스트: `lang="ko-KR" altLang="en-US"`
   - 영문 텍스트: `lang="en-US" altLang="ko-KR"`
   - 볼드: `b="1"` on `<a:rPr>`
   - 줄바꿈: 별도 `<a:p>` 요소 사용 (한 `<a:t>`에 다 넣지 말 것)
   - 빈 줄: `<a:p><a:endParaRPr sz="600"/></a:p>`
4. **클린/팩**: `python scripts/clean.py unpacked/` → `python scripts/office/pack.py unpacked/ output.pptx --original input.pptx`

### C. 크로스 프레젠테이션 슬라이드 복사

> **중요**: python-pptx의 OPC 레이어로는 프레젠테이션 간 슬라이드 복사가 안정적이지 않음. ZIP/XML 직접 조작 필요.

#### 프로세스:
1. 소스/타겟 PPTX를 각각 ZIP으로 풀기 (`zipfile.extractall`)
2. 소스 슬라이드 XML에서 `<p:cSld><p:spTree>` 내 shape 요소들을 타겟으로 복사
3. 소스 `.rels` 파일에서 이미지/미디어 관계(relationship) 확인
4. 미디어 파일을 타겟의 `ppt/media/`로 복사 (파일명 충돌 시 리네임)
5. 타겟 `.rels`에 새 relationship 추가 (새 rId 발급)
6. 복사된 XML 내 rId 참조를 새 rId로 업데이트
7. `[Content_Types].xml`에 새 미디어 확장자 등록
8. 디렉토리를 다시 ZIP으로 패킹

#### 주의사항:
- `nvGrpSpPr`, `grpSpPr`는 복사하지 않음 (타겟의 것 유지)
- 미디어 파일 크기가 같으면 동일 파일로 판단하여 재활용
- slideLayout, notesSlide 관계는 타겟의 것 유지

### D. 시각적 QA (품질 검증)

1. PPTX → PDF 변환: `soffice --headless --convert-to pdf`
2. PDF → 개별 이미지: `pdftoppm -jpeg -r 150 file.pdf output_prefix`
3. 수정된 슬라이드 이미지를 하나씩 확인
4. 체크 포인트: 텍스트 잘림, 이미지 누락, 레이아웃 깨짐, 폰트 렌더링
