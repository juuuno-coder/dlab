# 지식 베이스 (Knowledge Base)

NotebookLM 스타일 제안서 생성을 위한 참고 자료 저장소

## 📁 폴더 구조

### 1. `rfp-samples/` - 공고서/제안요청서/과업지시서
**용도**: 발주처 요구사항 패턴 학습, 평가 기준 추출

**넣을 자료**:
- 공고문 (PDF, HWP, DOCX)
- 제안요청서 (RFP)
- 과업지시서 (TOR)
- 사업계획서

**파일명 규칙**: `[발주처]_[사업명]_[연도]_공고서.pdf`
- 예: `부산시_봄꽃축제_2024_공고서.pdf`

---

### 2. `previous-proposals/` - 기존 제안서 (리멘/디어스 수행)
**용도**: 회사의 제안서 양식/톤앤매너/강점 학습

**넣을 자료**:
- 과거 제출한 제안서 (선정/탈락 모두)
- 발표 자료 (PPT)
- 결과 보고서

**파일명 규칙**: `[연도]_[사업명]_제안서_[결과].pdf`
- 예: `2024_청년작당소_제안서_선정.pdf`
- 예: `2023_문화행사_제안서_탈락.pdf`

---

### 3. `templates/` - 공통 양식/템플릿
**용도**: 회사 표준 템플릿, 재사용 가능한 섹션

**넣을 자료**:
- 회사 소개서
- 조직도 템플릿
- 안전관리계획 표준안
- 예산서 양식

**파일명 규칙**: `템플릿_[항목명].pdf`
- 예: `템플릿_회사소개.pdf`
- 예: `템플릿_안전관리.pdf`

---

### 4. `resources/` - 참고 자료
**용도**: 근거 자료, 통계, 벤치마크

**넣을 자료**:
- 유사 사업 결과보고서
- 통계 자료
- 선행 연구
- 뉴스 기사

**파일명 규칙**: `참고_[내용].pdf`
- 예: `참고_2024_축제_참가자통계.pdf`

---

## 🤖 AI 분석 프로세스

### Step 1: 자료 업로드
- 웹 관리자 페이지 또는 폴더에 직접 파일 추가

### Step 2: Gemini 2.0 Flash 멀티모달 분석
- 이미지 OCR (공고서 이미지, 스캔 문서)
- PDF 텍스트 추출
- 표/차트 데이터 구조화
- 핵심 키워드 추출

### Step 3: 멀티-에이전트 병렬 처리
- **Lead**: 전략/구조/KPI 도출
- **FE**: 시각화 요소 제안
- **UX**: 스토리텔링/흐름 설계
- **BE**: 데이터 근거 수집

### Step 4: 고품질 제안서 자동 생성
- 공통 양식 준수 (previous-proposals 학습)
- 발주처 맞춤 전략 (rfp-samples 기반)
- 근거 자료 자동 인용 (resources 활용)

---

## 📝 사용 방법

### 웹에서 사용
1. 관리자 페이지 → "지식 베이스" 메뉴
2. 파일 업로드 (드래그 앤 드롭)
3. 카테고리 자동 분류
4. Gemini API 자동 분석

### Claude Code에서 사용
```bash
/project:analyze-knowledge-base
/project:generate-proposal-from-kb [사업명]
```

## ⚠️ 주의사항

- **용량 제한**: 파일당 최대 20MB (Gemini API 제약)
- **보안**: 민감 정보 포함 시 `.gitignore`에 추가
- **포맷**: PDF, HWP, DOCX, 이미지(JPG/PNG) 지원
- **개인정보**: 개인정보 포함 문서는 익명화 후 업로드
